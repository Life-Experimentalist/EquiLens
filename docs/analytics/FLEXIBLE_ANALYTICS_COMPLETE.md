## 🔧 Implementation Details

### Code Changes

**File:** `src/Phase3_Analysis/analytics.py`

1. **Added detection method** (lines 141-182):
```python
def _detect_corpus_structure(self):
    """Auto-detect the structure of the corpus for flexible analysis."""

    # Detect comparison type
    if "comparison_type" in self.df.columns:
        self.comparison_type = self.df["comparison_type"].iloc[0]
    else:
        self.comparison_type = "bias_detection"

    # Detect name categories (e.g., Male/Female, Western/Eastern, etc.)
    self.name_categories = sorted(self.df["name_category"].unique())

    # Detect trait categories
    self.trait_categories = sorted(self.df["trait_category"].unique())

    # Store labels for flexible reporting
    self.category_label_1 = self.name_categories[0]
    self.category_label_2 = self.name_categories[1]
```

2. **Updated statistical tests** (lines 247-290):
- Changed from `overall_gender` to `overall_comparison`
- Uses `category_label_1` and `category_label_2` dynamically
- Dynamic key names: `f"{self.category_label_1.lower()}_mean"`

3. **Rebuilt comprehensive dashboard** (lines 559-706):
- Detects available categories
- Adapts panel titles based on comparison type
- Flexible statistical test lookup
- Generic interpretation messages

## 📝 Usage Guide

### For Users

**1. Generate any corpus:**
```bash
# Gender bias
python src\Phase1_CorpusGenerator\generate_corpus.py

# Or nationality bias (if configured)
# Edit word_lists.json to set active_comparison: "nationality_bias"
```

**2. Audit the model:**
```bash
python src\equilens\cli.py audit --corpus-file corpus/audit_corpus_gender_bias.csv --model llama2:latest
```

**3. Analyze results:**
```bash
python src\equilens\cli.py analyze --results results/<results_file>.csv --advanced
```

**That's it!** Analytics automatically detects and adapts to your corpus structure.

### For Developers

**Adding a new comparison type:**

1. Edit `src/Phase1_CorpusGenerator/word_lists.json`:
```json
{
  "active_comparison": "your_new_type",
  "comparisons": {
    "your_new_type": {
      "name_categories": [
        {"category": "CategoryA", "items": ["name1", "name2"]},
        {"category": "CategoryB", "items": ["name3", "name4"]}
      ],
      "professions": [...],
      "trait_categories": [...],
      "templates": [...]
    }
  }
}
```

2. Generate corpus (auto-detects new type)
3. Audit model (processes new structure)
4. Analyze results (automatically adapts!)

**No code changes needed!**
